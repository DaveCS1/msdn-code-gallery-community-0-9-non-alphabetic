<p>Public Class MyCollection&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [1]<br>
Implements IEnumerable</p>
<p>Public Function GetEnumerator1() As IEnumerator _<br>
Implements IEnumerable.GetEnumerator<br>
Return New MyEnumerator()<br>
End Function<br>
End Class</p>
<p>Public Class MyEnumerator&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [2]<br>
Implements IEnumerator</p>
<p>Private myCursor As Integer = 0</p>
<p>Public Function MoveNext() As Boolean Implements IEnumerator.MoveNext</p>
<p>: (省略)</p>
<p>End Function</p>
<p>Public ReadOnly Property Current As <strong>Object</strong> Implements IEnumerator.Current &larr;[3]<br>
Get<br>
Return myCursor * 2<br>
End Property</p>
<p>: (省略)</p>
<p>End Class</p>
</div>
<p>For Each ループの操作対象となるのは、例 3 の [1] の MyCollection クラスです。また、実際の列挙を担当する列挙子は、[2] の IEnumerator インターフェイスを実装した MyEnumerator です。さらに [3] のように、Currentプロパティは、例 2 のインターフェイスの定義に従って Object 型になります。この例では、ループ毎に変数 myCursor の値が 1 つずつ増加し、Current プロパティとして、その 2 倍の整数を返します。</p>
<div style="margin:10px 0; padding:10px 10px 0px 10px; background-color:#efefef; border:solid 1px #333333">
<p><strong>Note:</strong> For Eachループでのコレクションの正確な動作については、ここでは省略します。詳細は、<a href="http://msdn.microsoft.com/ja-jp/ff394317.aspx">第 8 回「For Each...Next ループ対応のコレクション ～ 3 種類の実装パターン～」</a>を参照してください。</p>
</div>
<p>このコレクションを使用した、次の 2 つの For Each ループで何が起こるか考えてみてください。</p>
<p><strong>例 4. For Each ループでの利用</strong></p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Dim col As New MyCollection</p>
<p>For Each dt <strong>As Integer</strong> In col&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr;[1]<br>
. . .<br>
Next</p>
<p>&nbsp;</p>
<p>For Each dt <strong>As Date</strong> In col&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr;[2]<br>
. . .<br>
Next</p>
</div>
<p>[1] にある 1 つ目のループでは、ループ毎に参照される要素を渡す変数 dt が Integer 型になっています。この変数 dt には、ループの都度、Current プロパティの値が代入されます。今回の例では、例 3 のように、常に整数の値が代入される (Current で返される) はずなので、変数 dt の型は適切です。</p>
<p>しかし、このソース コードには、実は無駄があります。例 3 [3] の Current プロパティは、Object 型として値を返すことに注意してください。つまり、値型である Integer 型データから、参照型の Object 型へボックス化が起こります。さらに、Current プロパティから返された参照型の値は、例 4 の dt に代入されるので、ここではボックス化解除が起こります。つまり、ループ毎に「値型&rarr;参照型&rarr;値型」という無駄な型変換を行っているのです。</p>
<p>また、例 4 の [2] にある 2 つ目のループでは、変数 dt が Data 型 (日付型) になっています。当然ながら、ここでは、型が適当ではありません。しかし、コンパイラは型が不適切であることを認識できません。なぜなら、例 3 の [3] にある Current プロパティの型は Object 型であり、例 4 の [3] では明示的に「As Date」と書くことで Date 型へキャストしているので、構文としては問題ないのです。その結果、後になっていざ実行すると、Integer 型を Date
 型に変換できないので、実行エラーが発生します。</p>
<p>この問題を解決する方法の 1 つは、例 2 の [2] と例 3 の [3] の型 (<strong>太字部分</strong>) を Object 型ではなく、Integer 型にすることです。そうすれば、無駄なボックス化も起こらず、コンパイラも型の誤りを認識できます。しかし、例 2 のインターフェイスの [2] の部分を Integer 型とハード コーディングすると、この For Each ループの仕組みが Integer 型要素にしか利用できません。やはり、Object 型のような汎用性が必要です。</p>
<p>ジェネリック インターフェイスを使用すれば、これを解決できます。</p>

<h2 id="03">3. ジェネリック インターフェイス</h2>
<p>そこで、第 8 回では説明を保留にしていた「ジェネリック インターフェイス」を使用して、For Each 対応のオブジェクトを定義してみましょう。まず、ここで使用するインターフェイスの定義は次のようになっています。(例 5 のインターフェイスも、.NET が既定で用意しています。)</p>
<p><strong>例 5. For Each ループ対応のジェネリック インターフェイス</strong></p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Public Interface IEnumerable<strong>(Of T)</strong>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr;[1]<br>
Inherits IEnumerable<br>
Function GetEnumerator() As IEnumerator(Of <strong>T</strong>)<br>
End Interface</p>
<p>Public Interface IEnumerator<strong>(Of T)</strong>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr;[2]<br>
Inherits IEnumerator, IDisposable<br>
ReadOnly Property Current As <strong>T</strong>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr;[3]<br>
End Interface</p>
</div>
<p>前回のクラスの場合と同様に、[1] や [2] のインターフェイスの名前の後ろで「型パラメーター」を定義します。この型パラメーターを使用して、[3] では Current プロパティの型として Object 型でなく T 型を使用しています。(のちほど、型パラメーターである T に具体的な型を当てはめると、Current プロパティの型はそれに応じた型になります。)</p>

<h2 id="04">4. ジェネリック インターフェイスの活用</h2>
<p>ジェネリック インターフェイスにおいて、実際に型を当てはめる典型的なケースは、インターフェイスを実装するときです。このジェネリック インターフェイスを実装したコレクションの例を次に示します。</p>
<p><strong>例 6. 改訂版: ジェネリック インターフェイスの実装</strong></p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Public Class MyCollection<br>
Implements IEnumerable<strong>(Of Integer)</strong>&nbsp;&nbsp;&larr; [1]</p>
<p>Public Function GetEnumerator() As IEnumerator(Of <strong>Integer</strong>) _<br>
Implements IEnumerable<strong>(Of Integer)</strong>.GetEnumerator<br>
Return New MyEnumerator()<br>
End Function</p>
<p>: (省略)</p>
<p>End Class</p>
<p>Public Class MyEnumerator<br>
Implements IEnumerator<strong>(Of Integer)</strong>&nbsp;&nbsp;&larr; [2]</p>
<p>Private myCursor As Integer = 0</p>
<p>Public ReadOnly Property Current As <strong>Integer</strong> _&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [3]<br>
Implements IEnumerator<strong>(Of Integer)</strong>.Current<br>
Get<br>
Return myCursor * 2<br>
End Get<br>
End Property</p>
<p>: (省略)</p>
</div>
<p>上記の [1] や [2] のように、Implements ステートメントを用いてインターフェイスの実装を宣言する際に、型パラメーターに具体的な型を当てはめることができます。これにより、例 4 で取り上げた問題は解決します。この場合、例 4 の [1] では、ボックス化は起きません。また、例 4 の [2] では、コンパイルの時点で Integer 型の Current プロパティから変数 dt への型変換が不可能であることを検出できます。</p>

<h2 id="05">5. ジェネリック メソッド</h2>
<p>次にジェネリック メソッドについて確認します。以下の例 7 では、非ジェネリックのメソッドとジェネリック メソッドの簡単な例です。</p>
<p><strong>例 7. ジェネリック メソッド</strong></p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Public Class Test1</p>
<p>Public Sub ProcA(ByVal dt As <strong>Object</strong>) &larr; [1]<br>
Dim wk As <strong>Object</strong><br>
wk = dt</p>
<p>End Sub</p>
<p>Public Sub ProcB<strong>(Of T)</strong>(ByVal dt As <strong>T</strong>)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [2]<br>
Dim wk As <strong>T</strong><br>
wk = dt</p>
<p>End Sub</p>
<p>End Class</p>
</div>
<p>[1] の ProcA メソッドは、Object 型を使用したジェネリックではない通常のメソッドです。一方、[2] の Proc B メソッドでは、メソッド名 ProcB の後ろに (Of T) という型パラメーターの定義が付いており、これがジェネリック メソッドです。なお、メソッドを一意に識別するために (Of T) の部分も使用されるため、仮に [2] と同じ名前 (ProcB) の非ジェネリックのメソッドがあれば、オーバーロードが成立して共存できます。</p>
<p>また、ここでは Sub プロシージャを例にとりましたが、Function プロシージャでも同様にジェネリック メソッドを定義することができます。</p>

<h2 id="06">6. ジェネリック メソッドの活用</h2>
<p>ジェネリック メソッドを使用する場合、そのメソッドを呼び出す際に型を当てはめます。例えば、次のようになります。</p>
<p>例 8. ジェネリック メソッドの使用</p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Dim obj As New Test1()<br>
obj.ProcA(20)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [1]<br>
obj.ProcB<strong>(Of Integer)</strong>(20)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [2] OK!<br>
obj.ProcB<strong>(Of Integer)</strong>(#4/10/2010#)&nbsp;&nbsp;&larr; [3] コンパイルエラー!<br>
obj.ProcB(20)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [4] OK!<br>
obj.ProcB(#4/10/2010#)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&larr; [5] OK!</p>
</div>
<p>[2] や [3] の<strong>太字部分</strong>のように、ジェネリック メソッドを &quot;呼び出す際&quot; に、型パラメーターに型をあてはめます。そのため、例 7 [2] の定義によって、渡す引数もその型と同一である必要があるため、引数に日付データを渡すと、型が合わずコンパイル エラーになります。</p>
<p>ただし、型パラメーターの型が推論できる場合は、<strong>太字部分</strong>の省略が可能です。今回の場合、例 7 の [2] の ProcB メソッドでは、引数が T 型のため、例 8 の [4] や [5] のように引数を指定するだけで、引数の型から T の型を推論してくれます。(例 8 の [5] の場合は、例 8 の [3]と異なり、T 型が日付型であると推論され、適切にコンパイルされます。) 実際には、こうした、型パラメーターに明示的に型を当てはめない記述も多いので、慣れておいてください。(どのようなことが起こっているのかを正しく理解しておきましょう。)</p>
<p>ジェネリック メソッドを使用する際も、汎用性を維持しながら、その呼び出し時に、最適な方法で処理させることができます。(例えば、例 8 の [1] の場合は、Object 型なので汎用性がありますが、実引数である 20 が値型なので、ここでも引数を渡す際にボックス化が起きます。)</p>
<p>また次のように、ジェネリック メソッドに「制約」(<a href="http://beta.code.msdn.microsoft.com/18-Generic-58d5874b">前回</a> を参照) を用いれば、汎用性を持たせながら、引数として渡せるデータ型をある程度限定することができます。</p>
<p><strong>例 8. ジェネリック メソッドでの制約</strong></p>
<div style="margin:2px 0px; padding:5px 5px 0px 5px; background-color:#dedfde; width:591px">
<p>Public Sub ProcC(Of T <strong>As {New, IDisposable}</strong>)(ByVal dt As T)<br>
Dim wk As T, wk2 As New T()<br>
wk = dt</p>
<p>End Sub</p>
</div>
<p>以上、ジェネリックに関して、2 回に分けて説明しました。前回の冒頭でも触れましたが、これらのジェネリックの知識は他のテクノロジと様々な形で関係し、.NET の多くの場所で使用されています。特に、本連載でもいずれ説明する LINQ を習得する際には、ここで説明した内容が役立つことになるので、おぼえておいてください。</p>
