<h2>はじめに</h2>
<p>ASP.NET Web サービスでは、様々な認&#35388; / 承認方式を使用することが可能です。<br>
今回の 10 行シリーズでは、その中から NetworkCredential クラスを使用した基本認&#35388;について、簡単なサンプル アプリケーションを通して解説します。<br>
なお、Web アプリケーションの基本的な開発方法についての説明は割愛しています。<br>
まず初めに「<a href="http://msdn.microsoft.com/ja-jp/events/dd279453.aspx">10 行でズバリ !! ASP.NET による Web サービスの開発 (VB.NET)</a>」および「<a href="http://msdn.microsoft.com/ja-jp/events/dd231294.aspx">10 行でズバリ !! Web サービス クライアント アプリケーションの開発 (VB.NET)</a>」をご覧ください。</p>
<p>&nbsp;</p>
<h2 id="webservice_auth_vb-2">作成するアプリケーションの概要</h2>
<p>今回のサンプルでは、 基本認&#35388;を使用する Web サービスと Web サービス クライアントの 2 つのアプリケーションを作成します。<br>
Web サービス クライアントでは、NetworkCredential クラスにより認&#35388;情報として ユーザー名およびパスワードを設定し Web サービスを呼び出します。<br>
呼び出された Web サービス側では、 認&#35388;済みのユーザー情報を取得し文字列として返します。</p>
<p>&nbsp;</p>
<h2 id="webservice_auth_vb-3">Web サービスの作成</h2>
<p>Visual Studio .NET 2003 を起動して、新規にプロジェクトを作成します。<br>
ここでは、[Visual Basic プロジェクト]で、[ASP.NET Web サービス] テンプレートを使用します。</p>
<p><img src="14813-image.png" alt=""> <br>
<strong>図 1. ASP.NET Web サービスの選択</strong></p>
<p>新しく追加された Web サービス （例では &quot;Service1.asmx&quot;） へコードを記述します。<br>
メニューより [表示] － [コード] を選択し、以下 Web メソッド Hello のコードを記述します。</p>
<p style="margin:20px 0px; padding:10px; background-color:#dedfde">&lt;WebMethod()&gt; _<br>
Public Function Hello() As String<br>
&nbsp;&nbsp;&nbsp; Return &quot;こんにちは &quot; &#43; Context.User.Identity.Name &#43; &quot; さん&quot;<br>
End Function</p>
<p>ここでは Context オブジェクトより、認&#35388;済みのユーザー名の情報を取得し文字列を追加したものを、Web メソッドの戻り値として返しています。</p>
<p>コードの記述が完了したら、 [ビルド] メニューより [WSAuth のビルド] を選択し、Web サービスのプロジェクトをビルドしておきます。</p>
<p>&nbsp;</p>
<h2 id="webservice_auth_vb-4">Web サービス クライアントの作成</h2>
<p>Web サービス クライアント として、今回は Windows アプリケーション を作成します。<br>
[ファイル] メニューの [プロジェクトの追加] [新しいプロジェクト] を選択し、[Visual Basic プロジェクト] で、[Windows アプリケーション] テンプレートを使用します。</p>
<p><img src="14814-image.png" alt=""> <br>
<strong>図 2. Windows アプリケーション の選択</strong></p>
<p>新しく追加された Windows フォーム （&quot;Form1.vb&quot;） に Text コントロール 2 つ、Button コントロール 1 つを配置し、以下の様にプロパティを設定します。</p>

この例では、Web 参照名 を WSAuth にて追加しています。</p>
<p>次に、Form1.vb デザイン画面の Button1 コントロールをダブルクリックし、以下の赤字で書かれているコードを記述します。</p>

次に、Hello メソッドを呼び出し、結果をメッセージボックスへ表示します。</p>

<h2 id="webservice_auth_vb-5">Web サービスへの基本認&#35388;の設定</h2>
<p>今回作成した Web サービスへ基本認&#35388;の設定を行います。<br>
スタートメニューの [管理ツール] より [インターネット インフォメーション サービス] を起動します。<br>
今回作成した Web サービス アプリケーション (この例では WSAuth) を右クリックし、メニューよりプロパティを選択します。</p>
<p><img src="14816-image.png" alt=""> <br>
<strong>図 4. Web サービス アプリケーションのプロパティ選択</strong></p>
<p>次に、[ディレクトリ セキュリティ] タブの [匿名アクセスおよび認&#35388;コントロール] - [編集] ボタンをクリックします。</p>
<p><img src="14817-image.png" alt=""> <br>
<strong>図 5. Web サービス アプリケーションのプロパティ</strong></p>
<p>デフォルトの認&#35388;方法では匿名アクセスが有効になっているため、ユーザー名およびパスワードによる認&#35388;が行われません。<br>
認&#35388;方法ダイアログにて、今回作成した Web サービス アプリケーションの認&#35388;方式として基本認&#35388;を使用するように設定します。</p>
<ul>
<li>匿名アクセス のチェックを外す </li><li>基本認&#35388; をチェック </li><li>統合 Windows 認&#35388; のチェックを外す </li></ul>
<p><img src="14818-image.png" alt=""> <br>
<strong>図 6. Web サービス アプリケーションの認&#35388;方法</strong></p>
<p>基本認&#35388;をチェックした際に、下記の警告ダイアログが表示されますが、ここでは [はい] ボタンをクリックします。</p>
<p><img src="14819-image.png" alt=""> <br>
<strong>図 7. 警告ダイアログ</strong></p>

<p>IIS にて SSL 通信を使用する場合は、サーバー&#35388;明書などの設定が必要となります。<br>
サーバー&#35388;明書などの詳細の設定については、下記の資料をご参照ください。</p>
<p><a href="http://www.microsoft.com/japan/msdn/security/iis-ssl/iis-ssl.aspx">セキュリティテスト用 SSL サーバーの構築 - ステップ バイ ステップ</a></p>
<p>なお、今回のサンプルプログラムにて SSL を使用する場合は、Web 参照 URL を http から https へ変更する必要があります。</p>

<h2 id="webservice_auth_vb-6">動作と解説</h2>
<p>サンプル プログラムを実行する前に、Windows アプリケーションをスタートアップ プロジェクトに設定します。<br>
ここでは、ソリューション エクスプローラ上にて Windows アプリケーションを選択後右クリックし、スタートアップ プロジェクトに設定を選択します。</p>
<p><img src="14821-image.png" alt=""> <br>
<strong>図 8. スタートアップ プロジェクト設定</strong></p>
<p>メニューより [デバッグ] － [開始] (または [F5] キー) を選択し、デバッグ実行をします。<br>
Windows アプリケーションが起動されるので、OS 上で利用可能なユーザー名およびパスワードを入力します。</p>
<p><img src="14822-image.png" alt=""> <br>
<strong>図 9. Windows アプリケーション</strong></p>
<p>ユーザー名およびパスワードを入力後、Hello ボタンをクリックします。<br>
Web サービスにて正しく認&#35388;が行われた場合は、下記のようなメッセージボックスが表示されます。</p>
<p><img src="14823-image.png" alt=""> <br>
<strong>図 10. メッセージ ボックス</strong></p>
<p>内部の動作としては、下記の順序にて基本認&#35388;が実施されています。</p>

<h2 id="webservice_auth_vb-7">おわりに</h2>
<p>このように、NetworkCredential クラスを使用した基本認&#35388;を使用することにより、Web サービスにて認&#35388;を行うことが可能です。<br>
今回ご紹介した基本認&#35388;以外の認&#35388;方式の情報については、下記の資料に記載されていますのでご参照ください。</p>
<p>ASP.NET を使用して作成した XML Web サービスのセキュリティ</p>
<p>Web サービスの場合も Web アプリケーションと同様に、認&#35388;情報を元にした URL 承認やプログラムによる承認などを実装することも可能ですので、この機会に是非チャレンジしてみて下さい。</p>
<p>&nbsp;</p>
<h2 id="webservice_auth_vb-8">補足情報</h2>
<p>Web サービスでは WS-Security というセキュリティ仕様が存在します。<br>
WS-Security では認&#35388;に関する仕様が定義されており、WS-Security の認&#35388;を使用することでも認&#35388;を行うことが可能です。</p>
<p><a href="http://www.microsoft.com/japan/msdn/webservices/security/understw.aspx">WS-Security 入門</a></p>
