<h2 id="01">1. はじめに</h2>
<p>データベースをアクセスするアプリケーションを構築する際、アプリケーションで必要なオブジェクト構造は、そのデータを&#26684;納しているデータベースのテーブル構造と必ずしも一致するとは限りません。そのため、データベースの読み書きには少なからずコーディングが必要になります。</p>
<p>ADO.NET Entity Framework は、このようなオブジェクトとデータベースのミスマッチ問題を解決するためのフレームワークであり、アプリケーションで必要となるオブジェクト構造とデータベースとの橋渡しの役割を担い、LINQなどによる汎用的なデータ アクセスの手段を提供します。</p>
<p>ここでは、Visual Studio を使用し、データベースのテーブル構造を基にした、アプリケーションで利用可能なオブジェクト構造であるエンティティ データ モデル (EDM) を作成します。さらには、作成した EDM に簡単なカスタマイズ (名前の変更やエンティティの統合など) を行っていきます。</p>
<p>なお、ここで説明する内容を実施するには、<strong>Visual Studio 2008 SP1 以上</strong>が必要になります。事前にインストールしておいてください。</p>
<p style="margin-top:20px"><a href="#top"><img src="10485-image.png" border="0" alt=""> ページのトップへ</a></p>
<hr>
<h2 id="02">2. EDM の作成準備</h2>
<p>ここでは EDM を使用するプロジェクトとして、コンソール アプリケーションのプロジェクトを使用します。まずはこれを新規作成しましょう。</p>
<p>Visual Studio を起動して、[ファイル] メニューの [新規作成] から [プロジェクト] をクリックし、[新しいプロジェクト] ダイアログを開きます。そして [プロジェクトの種類] で [Windows] を、[テンプレート] では [コンソール アプリケーション] を選択します。[プロジェクト名] は任意の名前を指定できますが、ここでは「EFApp」としました。</p>
<p><img src="10486-image.png" alt=""></p>
<p><strong>図 1 コンソール アプリケーションのテンプレートを選択してプロジェクトを新規作成</strong></p>
<p>[OK] ボタンをクリックすると、プロジェクトが作成されます。</p>
<p style="margin-top:20px"><a href="#top"><img src="10487-image.png" border="0" alt=""> ページのトップへ</a></p>
<hr>
<h2 id="03">3. データベースの準備</h2>
<p>今回はデータベースとして、<a href="http://www.microsoft.com/downloads/details.aspx?displaylang=en&FamilyID=06616212-0356-46a0-8da2-eebc53a68034" target="_blank">このページ (英語)</a> からダウンロードできる PUBS データベース ファイルを使用します。ダウンロードした PUBS.MDF ファイルを Windows エクスプローラーから、ソリューション エクスプローラー内のプロジェクト名部分にドラッグ
 &amp; ドロップしてください。これによりプロジェクトのあるフォルダーにファイルがコピーされます。データ ソース構成ウィザードが開きますが、これは不要ですので [キャンセル] ボタンをクリックして閉じます。これでデータベースの準備は完了です。</p>
<p style="margin-top:20px"><a href="#top"><img src="10488-image.png" border="0" alt=""> ページのトップへ</a></p>

<h2 id="04">4. EDM の追加</h2>
<p>次に、プロジェクトに EDM を追加します。これには、[プロジェクト] メニューの [新しい項目の追加] を実行して、[新しい項目の追加] ダイアログを開きます。そして、テンプレートで [ADO.NET Entity Data Model] を選択します。ファイル名は「Pubs」とします。</p>
<p><img src="10489-image.png" alt=""></p>
<p><strong>図 2 ADO.NET Entity Data Model のテンプレートを選択して項目を追加</strong></p>
<p>[追加] ボタンをクリックすると、Entity Data Model ウィザードが開始します。最初の画面では、[データベースから生成] が選択された状態で、[次へ] ボタンをクリックします。</p>
<p><img src="10490-image.png" alt=""></p>
<p><strong>図 3 Entity Data Model ウィザードの開始画面</strong></p>
<p>次に、基になるデータベースへのデータ接続を選択します。ここでは先ほどプロジェクトに追加した PUBS.MDF に関する設定が行われているのを確認し、[次へ] ボタンをクリックします。</p>
<p><img src="10491-image.png" alt=""></p>
<p><strong>図 4 Entity Data Model ウィザードでデータ接続を選択</strong></p>
<p>続いては EDM に含めるデータベースのテーブルを選択します。今回では、employee、pub_info、publishers の 3 つのテーブルを扱いますので、これらにチェックを入れます。</p>
<p><img src="10492-image.png" alt=""></p>
<p><strong>図 5 Entity Data Model ウィザードでテーブルを選択</strong></p>

<p>[完了] ボタンをクリックすると EDM が作成され、EDM デザイナーにより開かれます。このモデルには 3 つのエンティティが含まれているのが分かります。</p>

<h2 id="05">5. エンティティの内容確認</h2>
<p>ソリューション エクスプローラーを見ると、この EDM が Pubs.edmx で記述されているのが分かります (その下にある Pubs.Designer.cs は Pubs.edmx より自動生成される C# のクラス定義です)。これは XML ファイルであり、Pubs.edmx を右クリックし、[ファイルを開くアプリケーションの選択] を実行して、[XML エディター] を選択すれば、その記述内容を見ることができます。</p>
<p><img src="10495-image.png" alt=""></p>
<p><strong>図 7 XML で記述されたエンティティ データ モデルの内容</strong></p>
<p>この XML ファイルは、次の 3 つの主要な要素で構成されています。</p>
<ul>
<li>SSDL: ストア モデル。ストア スキーマ定義言語 (SSDL: Store Schema Definition Language) を使用して定義されています。データベースのテーブル構造やリレーションシップを記述しています。
</li><li>CSDL: 概念モデル。概念スキーマ定義言語 (CSDL: Conceptual Schema Definition Language) を使用して定義されています。これより開発者が利用可能なクラスが生成されます。
</li><li>C-S mapping: CSDL と SSDL のマッピング。マッピング スキーマ言語 (MSL: Mapping Schema Language) を使用して定義されています。概念モデルとストア モデルのマッピング (対応付け) を記述しています。
</li></ul>

<h2 id="06">6. EDM のカスタマイズ</h2>
<p>データベースを基に生成した EDM は、当然ながら、データベースのテーブルとほぼ同じ構造になっています。以降では、EDM をプログラムから利用しやすいようにカスタマイズしていきます。具体的には、名前の変更とエンティティの統合を行います。実際のテーブルとの対応付けを保った状態で EDM をカスタマイズできる点が ADO.NET Entity Framework のポイントです。</p>
<p>ソリューション エクスプローラーで Pubs.edmx をダブルクリックして、再度 EDM デザイナーを開きます。クラスの名前として複数形や小文字始まりは不適当ですので、まずはエンティティの名前を以下のように修正します。各エンティティの一番下にあるナビゲーション プロパティの名前も併せて修正してください。この修正は、EDM デザイナー上で名前の部分をクリックして行います。</p>
<ul>
<li>publishers &rarr; Publisher </li><li>employee &rarr; Employee </li></ul>
<p>修正後のエンティティは次の画面のようになります。</p>
<p><img src="10497-image.png" alt=""></p>
<p><strong>図 8 名前の修正を行ったエンティティ データ モデル</strong></p>
<p>ここでは行いませんが、スカラ プロパティの名前も自由に変更できます。なお、スカラ プロパティは各列のデータにアクセスするためのプロパティであり、ナビゲーション プロパティは、テーブル間のリレーションシップに基づき作成された、オブジェクト間の参照をたどるためのプロパティです。</p>
<p>次に、pub_info エンティティを Publisher エンティティに統合してみます。これにより、結果的に 1 つのエンティティが 2 つのテーブルにマッピングされることになります。</p>
<p>まず、pub_info エンティティをクリックして選択し、[Ctrl] キーを押しながら、スカラ プロパティである [logo] と [pr_info] を順にクリックして、これら 2 つを選択します。そして [Ctrl]&#43;[C] キーによりクリップボードにコピーします。続いて、Publisher エンティティをクリックして選択し、[Ctrl]&#43;[V] キーを押して貼り付けます。最後に、pub_info エンティティはもはや不要なので削除します。以上により、EDM デザイナーの画面は次のようになります。</p>
<p><img src="10498-image.png" alt=""></p>
<p><strong>図 9 pub_info エンティティを Publisher エンティティに統合</strong></p>
<p>次に、いま移動した 2 つのプロパティについて、マッピングを設定します。Publisher エンティティをクリックして選択し、Visual Studio の [表示] メニューの [その他のウィンドウ] から [マッピングの詳細] を実行します。</p>
<p><img src="10499-image.png" alt=""></p>
<p><strong>図 10 Publisher エンティティのマッピングについての詳細</strong></p>
<p>[マッピングの詳細] ウィンドウでは、Publisher エンティティにおいて、publishers テーブルの列 ([マッピングの詳細] ウィンドウ内の左側) が、エンティティの各プロパティ (同右側) とすでにマッピングされているのが分かります。</p>
<p>ここでは一番下にある &lt;テーブルまたはビューの追加&gt; の項目をクリックして、ドロップダウンから [pub_info] を選択します。これにより列のマッピングが (この場合には自動的に) に追加されます。</p>
<p><img src="10500-image.png" alt=""></p>
<p><strong>図 11 [マッピングの詳細] ウィンドウにおけるマッピングの追加</strong></p>
<p>これで Publisher エンティティは、publishers と pub_info の 2 つのテーブルにマッピングされていることになります。</p>
