<h2 id="masterpage_cs1">はじめに</h2>
<p>多くの Web サイトでは、ヘッダ、フッタ、メニューなど、複数の Web ページで共通的なデザインや機能を持っています。</p>
<p>ASP.NET 2.0 のマスタ ページは、このような共通部分をテンプレート化することができ、これによって各 Web ページで重複した実装を行う必要がなくなるとともに、修正箇所が 1 箇所に集約されるため保守性も向上します。</p>
<p>ここでは、このマスタ ページを活用した Web ページの共通化の方法について解説します。</p>

<p>ここでは、共通的なデザインを持つ 1 つのマスタ ページを作成し、2 つの Web ページでこれを共有します。</p>

<h2 id="masterpage_cs3">マスタ ページの作成</h2>
<p>共通デザインや機能を提供するマスタ ページを作成します。<br>
マスタ ページを作成するには、Visual Studio のメニューより、 [ファイル] － [新規作成] － [ファイル] を選択し、表示される [新しい項目の追加] ダイアログから、 [マスタ ページ] を選択します。</p>

<p>拡張子 &quot;.master&quot; のマスタ ページが追加されます。<br>
マスタ ページの作成方法は特に難しいことはなく、以降は、基本的に通常の Web ページを作成するのと同じ要領です。ドラッグ アンド ドロップで UI コントロールを配置し、UI コントロールをダブルクリックすることで処理ロジックを記述することができます。</p>
<p>各 Web ページ独自のコンテンツについては、ContentPlaceHolder コントロールを貼り付け、その表示領域を定義します。マスタ ページを作成した場合、デフォルトでひとつの ContentPlaceHolder コントロールが貼り付けられています。</p>
<p>今回は、デザイナ画面より ContentPlaceHolder コントロールの上部に、以下の実装を行います。</p>

<h2 id="masterpage_cs4">コンテンツ ページの作成</h2>
<p>各 Web ページ (コンテンツ ページ) を作成する場合、通常の Web ページと同様、 [Web フォーム] を選択します。</p>
<p>Visual Studio のメニューより [ファイル] － [新規作成] － [ファイル] を選択し、 [新しい項目の追加] ダイアログから [Web フォーム] を選択しますが、その際、ダイアログ下部の [マスタ ページを選択する] チェックボックスをチェックします。</p>
<p><img src="13875-image.png" alt=""></p>
<p><strong>図 4. コンテンツ ページの作成</strong></p>
<p>名前: Page1.aspx<br>
言語: Visual C#<br>
マスターページを選択する: チェックする</p>
<p>[追加] ボタンをクリックすると、 [マスタ ページを選択して下さい] ダイアログが表示されます。<br>
ここから、先ほど作成したマスタ ページを選択します。</p>
<p><img src="13876-image.png" alt=""></p>
<p><strong>図 5. マスタ ページの選択</strong></p>
<p>[OK] ボタンをクリックすると、Web ページが追加されます。<br>
マスタ ページのデザインがグレーアウトされた状態で表示され、最終的な Web ページのデザインを確認しながら、それぞれのコンテンツ ページの開発ができます。</p>
<p><img src="13877-image.png" alt=""></p>
<p><strong>図 6. コンテンツ ページのデザイン</strong></p>
<p>後は、ContentPlaceHolder コントロールで定義した領域に対して、通常の Web ページと同様、ドラッグ アンド ドロップで UI コントロールを配置し、UI コントロールをダブルクリックすることで処理ロジックを記述することができます。</p>
<p>ここでは、単純に 「ASP.NET 2.0 コンテンツ ページ」 という文字のみを記述します。</p>
<p><img src="13878-image.png" alt=""></p>
<p><strong>図 7. コンテンツ ページのデザイン (Page 1)</strong></p>
<h2 id="masterpage_cs5">動作確認</h2>
<p>それでは実行してみましょう。</p>
<p>Visual Studio のメニューより、 [デバッグ] － [デバッグなしで開始] を選択すると、ブラウザが起動され、コンテンツ ページ (Page1.aspx) が表示されます。</p>
<p>この際、マスタ ページのデザインと、実際のコンテンツ ページのデザインがマージされ、最終的にブラウザ上に表示されます。</p>
<p><img src="13879-image.png" alt=""></p>
<p><strong>図 8. 実行結果 (Page1.aspx)</strong></p>
<p>以上が、マスタ ページを利用した基本的な Web ページの作成方法となります。</p>

<p>ここでは、マスタ ページ上におかれたテキスト ボックスの値を、コンテンツ ページから取得する方法について解説します。</p>
<p>新しいコンテンツ ページ (Page2.aspx) を作成します。<br>

<h2 id="masterpage_cs7">おわりに</h2>
<p>マスタ ページを活用し、複数のWeb ページでデザインや機能を共通化する方法について解説しました。</p>
<p>マスタ ページを使わずにそれぞれのページごとに共通部分を実装した場合、変更が発生すると、すべてのページを修正しなければなりません。これは、手間がかかる上に、修正漏れが発生するといった危険性もあります。</p>
<p>マスタ ページによって、よりシンプル、かつ直感的に共通化を行うことができ、これによって、</p>
<ul>
<li>冗長な機能やデザインを作成したり、修正するための手間がなくなる </li><li>共通機能と個別機能を分けて並行開発できる </li><li>既に実績ある共通部品を再利用することによって品質の向上を期待することができる </li></ul>
<p>などのメリットが考えられます。</p>

<p>マスタ ページの基本的な作成方法について解説してきました。</p>
<p>このように、基本的には Visual Studio が必要な定義や作業を自動的に行うため、マスタ ページの仕組みについて特に意識しなくても開発は可能です。しかし、より緻密な制御を行う場合には、やはりマスタ ページの仕組みについて理解する必要があるため、ここで補足事項として触れます。</p>
<p>まず、マスタ ページについてみてみましょう。</p>
<p>Visual Studio を [ソース] ビューに切り替え、マスタ ページの HTML ソースを確認します。<br>
通常の Web フォームと同じような HTML ソースが定義されていますが、ポイントが 2 点あります。</p>
<ol>
<li>まず、先頭行に Master ディレクティブ (&lt;%@Master ...%&gt;) があることを確認することができます。
<table cellpadding="5">
<tbody>
<tr>
<td bgcolor="#333333">&nbsp;<span style="color:#ffffff">&lt;%@Master Language=&quot;C#&quot; CodeFile=&quot;MasterPage.master.cs&quot; Inherits=&quot;MasterPage&quot; %&gt;</span></td>
</tr>
</tbody>
</table>
<br>
Master ディレクティブはこのページがマスタ ページであることを示し、マスタ ページに関する設定を行うことができます。 </li><li>さらに、本文中に ContentPlaceHolder コントロール (&lt;asp:ContentPlaceHolder&gt;) があることを確認することができます。<br>
<table cellpadding="5">
<tbody>
<tr>
<td bgcolor="#333333">&nbsp;<span style="color:#ffffff">&lt;asp:ContentPlaceHolder ID=&quot;ContentPlaceHolder1&quot; runat=&quot;server&quot;%&gt;</span></td>
</tr>
</tbody>
</table>
</li><li><br>
ContentPlaceHolder コントロールは、いわばコンテンツを表示するための予約域です。実際にはこの ContentPlaceHolder コントロールは 1 つのマスタ ページの中に複数定義することができます。
</li></ol>
<p>次に、コンテンツ ページについてみてみましょう。</p>
<p>Visual Studio を [ソース] ビューに切り替え、コンテンツ ページの HTML ソースを確認します。<br>
コンテンツ ページについてはポイントが 3 点あります。</p>
<ol>
<li>通常の Web ページとは異なり、&lt;html&gt;、&lt;head&gt;、&lt;body&gt; タグは定義しません。実際のWeb ページ表示時には、マスタ ページの定義が適用されます。
</li><li>Page ディレクティブ (&lt;%@Page ... %&gt;) のMasterPageFile にマスタ ページ ファイル (.master) の場所を指定します。<br>

マスタ ページ上の予約域 (ContentPlaceHolder コントロール) との対応づけのために、Content コントロールの ContentPlaceHolderID に、マスタ ページの該当する ContentPlaceHolder コントロールの ID を指定します。<br>
